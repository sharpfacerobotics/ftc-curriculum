# Unit 1: Choosing Your Programming Tool

:::info Prerequisites
No prior programming experience required. Just bring curiosity.
:::

---

## What You'll Learn

By the end of this unit you'll be able to:

- Name the three FTC programming tools and what makes each one different
- Explain when you'd choose one tool over another
- Understand where your code actually runs on the robot

---

## Explanation

FTC gives you three ways to program your robot. They all produce the same result — a robot that moves — but they're designed for different skill levels and workflows.

### Blocks

Blocks is a **visual, drag-and-drop** programming environment that runs in a web browser connected to your Control Hub. You don't type any code — instead you snap together puzzle pieces that represent commands. It's the fastest way to get a robot moving, and it requires zero prior programming experience.

**Best for:** First-time programmers, quick prototyping, or younger team members getting started.

**Limitation:** As your robot gets more complex, Blocks becomes harder to manage. There's no easy way to use advanced Java features.

---

### OnBot Java

OnBot Java is a **text-based editor** that also runs in a web browser connected to the Control Hub. You write actual Java code, and it compiles and deploys directly on the robot — no laptop software needed beyond a browser.

**Best for:** Teams learning Java for the first time, or anyone who's outgrown Blocks but doesn't want to set up Android Studio.

**Advantage over Blocks:** Full access to Java features — loops, classes, libraries.

**Advantage over Android Studio:** No installation required. Works from any device on the same WiFi network as the robot.

---

### Android Studio

Android Studio is a **professional-grade IDE** (Integrated Development Environment) — the same tool used by Android app developers. It runs on a dedicated laptop, not in a browser. It offers advanced features like a full debugger, code completion, and version control integration.

**Best for:** Advanced teams, or anyone writing complex autonomous code with vision processing and pathing libraries.

**Limitation:** Requires a dedicated laptop at the field, more setup time, and a steeper learning curve.

---

### Quick Comparison

| Feature | Blocks | OnBot Java | Android Studio |
|---|---|---|---|
| Requires coding? | No | Yes (Java) | Yes (Java) |
| Runs in browser? | Yes | Yes | No |
| Needs a laptop? | No | No | Yes |
| Advanced debugging? | No | Limited | Full |
| Best for | Beginners | Learners | Experts |

**Q1. Which tool is described as "the fastest way to get started"?**

**B — Blocks.** It requires no prior coding experience and runs directly in the browser.

</details>

---

**Q2. Which tool is recommended only for advanced users?**

<details>
<summary>Show answer</summary>

**B — Android Studio.** It requires a dedicated laptop, Java knowledge, and significantly more setup.

</details>

---

**Q3. Do you need a web browser to use OnBot Java?**

<details>
<summary>Show answer</summary>

**A — Yes.** OnBot Java is entirely browser-based. You access it by connecting to the Control Hub's WiFi network and navigating to its IP address.

</details>

---

**Q4. What is the key limitation of Blocks compared to OnBot Java?**

<details>
<summary>Show answer</summary>

Blocks doesn't give you access to the full Java language. As robot code grows more complex — custom classes, advanced math, vision processing — Blocks becomes difficult or impossible to work with.

</details>
