Lesson 5: Loops

Explanation​

Why Loops?​

A TeleOp program needs to read gamepad input thousands of times per second. An autonomous program needs to keep moving until a sensor threshold is hit. You can't write that by hand — you need code that repeats automatically. That's what loops do.

The while Loop​

A while loop repeats its block as long as its condition is true. The moment the condition becomes false, the loop exits and code continues after it.

while (condition) {
    // runs repeatedly
}

In FTC, the most important while loop is this one:

while (opModeIsActive()) {
    // your TeleOp code lives here
    telemetry.update();
}

opModeIsActive() returns true while the OpMode is running and false the moment the driver presses STOP. This is the heartbeat of every TeleOp OpMode.

Infinite loops

If the condition never becomes false, your loop runs forever — freezing the robot. Always make sure your condition can eventually be false, or that opModeIsActive() will eventually return false.

The for Loop​

A for loop is used when you know exactly how many times to repeat. It combines the counter variable, the condition, and the increment into one line.

for (int i = 0; i < 5; i++) {
    telemetry.addLine("Loop iteration: " + i);
    telemetry.update();
    sleep(500); // wait 500ms
}

Breaking down for (int i = 0; i < 5; i++):

PartMeaning
int i = 0Start the counter at 0
i < 5Keep looping while i is less than 5
i++Add 1 to i after each iteration

This runs 5 times (i = 0, 1, 2, 3, 4). When i reaches 5 the condition i < 5 is false and the loop exits.

sleep() — Pausing in Autonomous​

In a LinearOpMode, sleep(milliseconds) pauses execution for a set amount of time. It's commonly used in autonomous to wait for a motor to finish moving before issuing the next command.

// Drive forward for 1 second
leftMotor.setPower(0.5);
rightMotor.setPower(0.5);
sleep(1000);  // wait 1000ms = 1 second

// Stop
leftMotor.setPower(0.0);
rightMotor.setPower(0.0);
Time-based autonomous is unreliable

Battery voltage, carpet friction, and wheel slip all affect how far the robot actually moves in a given time. sleep() is fine for testing but encoder-based movement (covered in Unit 2) is far more reliable for competition.

break and continue​

Two keywords that modify loop behavior:

break — immediately exits the loop no matter what:

while (opModeIsActive()) {
    if (distanceSensor.getDistance() < 10) {
        break; // exit the loop when close enough
    }
    driveForward();
}

continue — skips the rest of the current iteration and jumps to the next one:

for (int i = 0; i < 10; i++) {
    if (i == 5) {
        continue; // skip iteration 5
    }
    telemetry.addData("i", i);
    telemetry.update();
}

Nested Loops​

Loops can live inside other loops. The inner loop completes fully for each iteration of the outer loop:

for (int pass = 0; pass < 3; pass++) {
    for (int step = 0; step < 4; step++) {
        telemetry.addData("Pass", pass);
        telemetry.addData("Step", step);
        telemetry.update();
        sleep(200);
    }
}
// This runs 3 × 4 = 12 total iterations

Quiz​

Q1. What does opModeIsActive() return when the driver presses STOP?

Show answer

false — which causes the while (opModeIsActive()) loop to exit, ending the OpMode gracefully.

Q2. How many times does this loop run?

for (int i = 0; i <= 5; i++) { }
Show answer

6 times — i goes through 0, 1, 2, 3, 4, and 5. The condition is i <= 5 (less than or equal), so it includes 5. A common mistake is thinking i < 5 and i <= 5 are the same.

Q3. What is the danger of using sleep() for autonomous distance control?

Show answer

The distance the robot travels in a given time depends on battery voltage, friction, and floor conditions — all of which vary. A routine that works perfectly at full battery might fall short when the battery is half-drained. Encoder-based control is consistent regardless of these factors.

Q4. What does break do inside a loop?

Show answer

It immediately exits the loop, skipping any remaining iterations and any code below it inside the loop body. Execution continues with whatever comes after the closing brace of the loop.
