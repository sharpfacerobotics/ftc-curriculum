Lesson 1: Choosing Your Programming Tool

No experience needed

This lesson assumes zero prior programming knowledge. Start here.

Explanation​

FTC gives you three ways to program your robot. They all produce the same result — a working robot — but they're built for different skill levels and workflows. Picking the right tool means less frustration and more time actually building.

Blocks​

Blocks is a visual, drag-and-drop programming environment that runs in a web browser. You connect your laptop to the Control Hub's WiFi network, open the browser, and snap together puzzle pieces that represent commands — no typing required. It compiles and deploys your code to the robot automatically.

Blocks is the fastest way to get a robot moving. It's designed specifically for people with no prior programming experience, and it hides a lot of Java complexity behind friendly visual elements.

Best for: First-year programmers, quick prototypes, or anyone who wants results immediately.

Limitation: As your code grows more complex, Blocks becomes hard to manage. There's no clean way to write custom classes, use external libraries, or implement advanced autonomous pathing.

OnBot Java​

OnBot Java is a text-based Java editor that also runs in a web browser connected to the Control Hub. Instead of dragging blocks, you type actual Java code. The editor compiles it directly on the Control Hub and pushes it to the robot — no additional software needed beyond a browser.

This is the natural step up from Blocks. You get the full power of Java without needing to install anything on your laptop.

Best for: Teams learning Java for the first time, or anyone who has outgrown Blocks.

Advantage over Blocks: Full access to Java — loops, classes, custom methods, math libraries.

Advantage over Android Studio: No installation, works from any device on the same WiFi network as the robot.

Android Studio​

Android Studio is a professional IDE (Integrated Development Environment) — the same tool used by Android app developers worldwide. It runs as installed software on a dedicated laptop, not in a browser. It gives you a full debugger, intelligent code completion, refactoring tools, and deep integration with version control like Git.

Most top FTC teams use Android Studio for their final competition code. It's the most powerful option but also the most complex to set up and learn.

Best for: Advanced teams writing complex autonomous routines, vision processing, or external pathing libraries like Pedro Pathing.

Limitation: Requires a dedicated laptop at the field, more initial setup, and a solid foundation in Java before you'll feel comfortable.

Quiz​

Q1. Which tool requires no prior coding experience and works entirely through drag-and-drop?

Show answer

Blocks. It's designed specifically for beginners and hides all Java syntax behind visual puzzle pieces.

Q2. You're at a competition and need to quickly tweak your autonomous code from a Chromebook. Which tool do you use?

Show answer

OnBot Java. It runs in any browser connected to the Control Hub's WiFi — no laptop software needed.

Q3. Which tool is the only one that offers a full debugger?

Show answer

Android Studio. Its debugger lets you pause code mid-execution, inspect variable values, and step through your program line by line.

Q4. True or false: You can use Android Studio from any browser connected to the robot.

Show answer

False. Android Studio is installed software that runs on a dedicated laptop. Only Blocks and OnBot Java are browser-based.
